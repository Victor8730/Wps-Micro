# Security Policy

## Supported Versions

| Version | Security support |
| --- | --- |
| 3.x | Supported |
| 2.1.x | Critical security fixes only |
| Earlier versions | Not supported |

Only the latest patch release in a supported line receives fixes.

## Reporting A Vulnerability

Do not open a public issue for a suspected vulnerability.

Use GitHub private vulnerability reporting from the repository Security tab.
If that option is unavailable, email `victor8730@gmail.com` with:

- the affected version;
- steps to reproduce the issue;
- the expected security impact;
- any suggested mitigation.

Avoid including credentials, personal data, or production secrets in the
report. Please allow time to investigate and prepare a coordinated fix before
publishing technical details.
